# OpenAPI_project

## 프로젝트 개요

서울시 버스 경로 OpenAPI를 Python Requests로 한 번 호출하는 최소 예제입니다. 소스 코드에 고정된 출발·도착 좌표를 HTTPS GET 요청으로 전송하고 응답 본문의 원시 바이트를 표준 출력에 표시합니다.

## 현재 구현 기능

- `SEOUL_OPENAPI_SERVICE_KEY` 환경 변수 읽기
- 환경 변수가 없거나 비어 있으면 `RuntimeError` 발생
- 서울시 버스 경로 API에 고정 좌표 전송
- HTTPS GET, 30초 타임아웃과 HTTP 오류 검사
- `response.content` 원시 바이트 출력
- 환경 변수 및 요청 구성에 대한 모의 단위 테스트 2개

응답 파싱, 경로 가공, 사용자 입력, 파일·데이터베이스 저장과 웹 UI는 구현되어 있지 않습니다.

## 기술 스택

- Python
- Requests `>=2.31,<3`
- `unittest`와 `unittest.mock`

지원 Python 버전은 저장소에 명시되어 있지 않습니다.

## 설치

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -r requirements.txt
```

현재 검증 환경의 Requests 2.31.0은 `requirements.txt`의 선언 범위를 충족합니다.

## 실행

> 과거 원격 이력에 서비스키로 의심되는 문자열 흔적이 있습니다. 보안 주의 사항의 수동 조치가 완료되기 전에는 저장소를 공개하거나 실제 API를 호출하지 마세요.

수동 보안 조치 이후 새로 발급한 인증값을 안전한 방식으로 환경 변수에 전달합니다.

```bash
export SEOUL_OPENAPI_SERVICE_KEY='<인증값>'
python3 test.py
```

실제 인증값을 코드, README, 셸 출력 또는 Git이 추적하는 설정 파일에 기록하지 마세요. 운영 환경에서는 셸 기록에 인증값이 남지 않는 비밀 관리 방식을 사용해야 합니다.

실행 시 현재 코드에 고정된 좌표를 사용해 외부 API 요청 1회를 보냅니다. 좌표가 의미하는 실제 출발지와 도착지는 저장소 문서에서 확인되지 않았습니다.

## 테스트

```bash
python3 -m unittest -v test_openapi.py
```

현재 두 테스트는 다음 항목을 확인합니다.

- 서비스키 환경 변수가 없을 때 오류가 발생하는지
- 모의 HTTP 세션에 API URL과 30초 타임아웃을 전달하고, 요청 매개변수에 `startX`가 포함되며 HTTP 오류 검사를 호출하는지

현재 검증 환경에서 두 테스트가 통과했습니다. 외부 HTTP 요청은 모의 처리하므로 실제 인증, API 연결, 응답 형식, 서비스 제한 또는 비용을 검증하지 않습니다.

## 데이터 저장 방식

데이터베이스와 파일 저장을 사용하지 않습니다. API 응답을 파싱하지 않고 원시 응답 본문을 현재 프로세스의 표준 출력으로 내보냅니다.

## 보안 주의 사항

> **중요:** 현재 HEAD가 환경 변수를 사용한다는 사실만으로 저장소의 인증정보 노출 위험이 해결된 것은 아닙니다.

`origin/master`가 가리키는 과거 커밋에는 서비스키로 의심되는 문자열 리터럴이 추적된 흔적이 있습니다. 실제 값은 이 README에 기록하지 않습니다.

해당 값이 실제 서비스에 사용되었다면 저장소 소유자 또는 승인된 관리자가 다음 조치를 수행해야 합니다.

- 기존 서비스키 폐기 및 재발급
- 서비스 사용·접근 기록과 비정상 호출 여부 확인
- API 할당량 또는 비용 영향 확인
- 인증값을 Git이 추적하지 않는 안전한 설정으로 관리
- 조직 승인과 협업자 조율을 거친 원격 Git 이력 정리
- 정리 후 현재 파일과 전체 이력에 인증값이 남아 있지 않은지 재검증

이 README 정비에서는 서비스키 값을 기록하지 않았으며 원격 push나 Git 이력 재작성도 수행하지 않았습니다.

## 알려진 문제 및 제한 사항

- 출발·도착 좌표가 소스 코드에 고정되어 있고 위치 의미가 문서화되어 있지 않습니다.
- 응답 상태와 스키마를 해석하거나 사용자 친화적으로 표시하지 않습니다.
- 실제 API와 네트워크 연결은 검증되지 않았습니다.
- 서비스 호출 제한, 이용 조건과 비용 여부가 확인되지 않았습니다.
- 실제 API 호출을 다루는 통합 테스트가 없습니다.
- 운영 또는 배포 구성이 없습니다.

## 포트폴리오 전환 검토

**분류: F. 수동 확인 필요**

현재 구현은 환경 변수 처리와 고정 좌표를 이용한 서울 버스 API 1회 호출을 보여 주는 최소 예제입니다. 원시 응답만 출력하고 파서, 사용자 흐름, 저장 기능과 실제 API 검증이 없어 현 상태의 전시 가치는 제한적입니다.

원격 및 과거 이력에 서비스키로 의심되는 문자열 흔적이 있으므로 저장소를 공개하거나 실제 API를 호출해서는 안 됩니다. 저장소 소유자 또는 승인된 관리자가 기존 키 폐기·재발급, 서비스 사용·접근 기록과 비용 영향 확인, 승인된 원격 이력 정리와 전체 재검증을 완료한 뒤 좌표의 목적과 원래 학습 목표를 확인하고 구현 재개, 코드 보존 또는 폐기 검토 중 하나로 다시 분류해야 합니다.
